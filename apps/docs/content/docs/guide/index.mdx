Use this section when you are adding Pipr to a repository, changing how it reviews pull requests, or debugging what it publishes.

## First run

Follow this path for the first successful pull request review.

1. Install the CLI from the GitHub Release installer.
2. Run `pipr init` in the repository root.
3. Add the provider secret used by `.pipr/config.ts`.
4. Review the generated GitHub workflow and commit the files.
5. Open or update a pull request and check the Main Review Comment.
6. Use local review commands while tuning config.
